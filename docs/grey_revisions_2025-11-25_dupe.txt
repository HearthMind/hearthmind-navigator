2. Wire Benefits/Resources to the config files

Assuming you’ve got something like config/benefits.yaml and config/resources.yaml, you can do:

In app.py (or wherever your routes are):

import yaml
from flask import Flask, render_template

app = Flask(__name__)

with open("config/benefits.yaml", "r") as f:
    BENEFITS = yaml.safe_load(f)

with open("config/resources.yaml", "r") as f:
    RESOURCES = yaml.safe_load(f)

@app.route("/benefits")
def benefits():
    return render_template("benefits.html", benefits=BENEFITS)

@app.route("/resources")
def resources():
    return render_template("resources.html", resources=RESOURCES)


Then in benefits.html:

{% extends "base.html" %}
{% block content %}
<section class="section">
  <h1>Benefits Guide</h1>
  <p>No judgment. Just what you qualify for and how to get it.</p>

  <div class="card-grid">
    {% for item in benefits %}
      <article class="card">
        <h2>{{ item.name }}</h2>
        <p>{{ item.description }}</p>
        {% if item.link %}
          <a href="{{ item.link }}" target="_blank">Learn more</a>
        {% endif %}
      </article>
    {% endfor %}
  </div>
</section>
{% endblock %}


Same idea for resources.html: loop over resources.

Navigator becomes data-driven: you edit YAML/JSON, not HTML.
